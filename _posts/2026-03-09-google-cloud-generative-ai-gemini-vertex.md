## 🌟 Google Cloud Generative AI

**GoogleCloudPlatform/generative-ai** is the official repository for Generative AI on Google Cloud, featuring comprehensive notebooks, code samples, and resources for building AI workflows with **Gemini on Vertex AI**.

Just released: **[Gemini 3.1 Pro](https://cloud.google.com/vertex-ai/generative-ai/docs/models/gemini/3-1-pro)** — the latest iteration of Google's flagship multimodal model.

### What's Inside

This repository serves as a one-stop shop for developers working with Google's generative AI stack:

| Section | Purpose |
|---------|---------|
| **[gemini/](https://github.com/GoogleCloudPlatform/generative-ai/tree/main/gemini)** | Starter notebooks, use cases, function calling, and sample apps for Gemini |
| **[search/](https://github.com/GoogleCloudPlatform/generative-ai/tree/main/search)** | Vertex AI Search — build rapid search engines for websites and enterprise data |
| **[rag-grounding/](https://github.com/GoogleCloudPlatform/generative-ai/tree/main/rag-grounding)** | Retrieval Augmented Generation (RAG) and Grounding with Vertex AI |
| **[vision/](https://github.com/GoogleCloudPlatform/generative-ai/tree/main/vision)** | Imagen on Vertex AI — image generation, editing, captioning, and visual Q&A |
| **[audio/](https://github.com/GoogleCloudPlatform/generative-ai/tree/main/audio)** | Chirp (Google's Universal Speech Model) for speech-to-text on Vertex AI |
| **[setup-env/](https://github.com/GoogleCloudPlatform/generative-ai/tree/main/setup-env)** | Environment setup for Google Cloud, Vertex AI SDK, Colab, and Workbench |

### Getting Started

**Prerequisites:**
- Google Cloud Platform account
- Vertex AI API enabled
- Python 3.8+ or Node.js (depending on samples)

**Quick Start:**

```bash
# Clone the repository
git clone https://github.com/GoogleCloudPlatform/generative-ai.git
cd generative-ai

# Follow setup instructions
cd setup-env

# Choose your environment (Colab or Vertex AI Workbench)
```

Most notebooks run directly in **Google Colab** with zero setup, making it easy to experiment before committing to infrastructure.

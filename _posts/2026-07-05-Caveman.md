Why use many token when few do trick? That's the delightfully simple premise behind **Caveman**, a skill/plugin for AI coding agents that slashes output tokens by 65% simply by making your AI talk like a prehistoric ancestor. Built by independent developer Julius Brussee, Caveman has exploded to over 83,000 stars in just three months by tapping into a universal developer pain point: AI assistants are pathologically verbose, burning through token budgets with meandering explanations and unnecessary courtesies.

Caveman is a lightweight JavaScript skill that rewrites the agent's system prompt to enforce concise, caveman-style speech — while preserving code, commands, and error output byte-for-byte. Instead of "I'll now proceed to create a new file called utils.js with the following content," the agent says "make file utils.js." The architecture is remarkably portable: a single SKILL.md file is distributed across `.claude-plugin`, `.codex`, and other agent-specific directories via automated sync, with a unified installer (`bin/install.js`) that handles JSONC settings across platforms. It works with Claude Code, Codex, Gemini, Cursor, Windsurf, Cline, Copilot, and 30+ other agents.

What sets Caveman apart is its layered approach. Users can choose from multiple "caveman levels" ranging from mild terseness to full Cro-Magnon minimalism. The project includes `/caveman` slash commands for Claude Code, comprehensive benchmarks showing consistent 65% token reduction across diverse coding tasks, and a growing plugin ecosystem. With 252 commits, 4,700+ forks, and a thriving community of 155+ open issues and 201 pull requests, Caveman has evolved well beyond a meme into a genuinely useful optimization tool for anyone tired of paying for filler words.

Caveman's meteoric rise reflects a broader shift in the AI developer community: as token costs add up and context windows fill with noise, the pendulum is swinging from "be thorough and helpful" toward "just give me the answer." It's a reminder that sometimes the most impactful optimization isn't a new model or algorithm — it's just telling the AI to shut up and get to the point.

When developer Hasan Eyldrm set out to build a workout application in early 2026, he ran into the same wall every fitness-app builder hits: there was no clean, structured, open exercise dataset that just worked. ExerciseDB v1 existed but lacked multilingual support, developer tooling, and a friendly onboarding experience. Rather than build yet another half-baked scraper, Eyldrm took ExerciseDB's 1,324-exercise foundation — sourced from AscendAPI via a Kaggle re-host — and layered on six-language instruction translations with a complete developer setup wizard that generates database schemas, API client code in seven languages, and structured LLM prompts in a single browser tab. The result is exercises-dataset, a meticulously curated launchpad that turns the first three days of fitness-app boilerplate into a ten-minute job, which is exactly why it hit #1 trending today.

Built as a flat JSON file with companion static HTML tools, the dataset's architecture is elegant in its simplicity. The core file, data/exercises.json, is a 1,324-element array where each exercise carries a unique numeric ID, category and body part classification, equipment requirement, primary and secondary muscle targets, and the critical innovation — a nested instructions object with full step-by-step guidance in English, Spanish, Italian, Turkish, Russian, and Chinese. The repository ships two standalone HTML applications that run with zero server dependencies: index.html provides an infinite-scroll, filterable exercise browser with live search across all six languages, while setup.html is the developer command center — it generates CREATE TABLE SQL for SQL Server, PostgreSQL, MySQL, and SQLite, produces ready-to-paste client code in JavaScript, Python, C#, Java, PHP, Go, and cURL, and even scaffolds a structured prompt you can feed to ChatGPT, Claude, or Gemini to generate a complete production REST API in Express.js, FastAPI, ASP.NET Core, Spring Boot, Laravel, or Gin. The dataset's breadth is what makes it immediately production-useful. Upper arms lead the body-part distribution with 292 exercises, followed by upper legs (227), back (203), waist (169), and chest (163), while equipment coverage spans body weight (325 exercises — roughly 25% of the dataset), dumbbell (294), cable (157), barbell (154), and twelve other categories from kettlebell to Smith machine. The sample exercises section reads like a greatest-hits of strength training: barbell bench press with scapular retraction cues, the deadlift with bracing and hinge mechanics, full squats with high-bar versus low-bar positioning, and pull-ups with lat-engagement initiation. The setup wizard is where the project truly shines for developers — select your preferred database from the dropdown, hit generate, and you get a complete INSERT-heavy SQL file built entirely client-side with JavaScript, alongside auto-updating code snippets that swap your custom API base URL into every example. The LLM prompt feature goes further: pick Express.js + PostgreSQL, paste the generated prompt into any major AI coding assistant, and it will produce a working REST API with CRUD endpoints, filtering, pagination, and multilingual support in a single response.

exercises-dataset is trending because it solves a genuinely painful developer experience problem with refreshing pragmatism. The fitness-tech space is booming — workout trackers, AI-powered coaching, rehabilitation apps, and corporate wellness platforms all need exercise data — yet until now there was no turnkey, multilingual, developer-friendly dataset that didn't require API keys, rate limits, or expensive licensing. Eyldrm's approach of shipping a pure data file with build-your-own-backend tooling sidesteps vendor lock-in and gives developers complete control over their stack. The recent addition of Chinese language support and the Claude-assisted commit history also reflect a project that is iterating fast and genuinely global in ambition.
